# Comunicação (visão geral)

A área de **Comunicação** centraliza todo o fluxo de e-mails do sistema: os modelos de mensagem (templates), o registro de tudo que foi disparado (histórico) e as integrações que avisam sistemas externos a cada evento (webhooks). É o ponto de partida para configurar e acompanhar a comunicação automática da sua leiloeira.

## Como acessar

**Plataforma** → **Comunicação**

![Comunicação](/img/manual/erp/comunicacao.png)

## O que você vê nesta tela

A tela é um painel (hub) com três cartões, cada um levando a uma seção. Clique no cartão para abrir a seção correspondente.

| Cartão | Para que serve |
|---|---|
| **Templates** | Modelos reutilizáveis de e-mail usados em toda a plataforma: boas-vindas, cobrança, notificações, confirmações. |
| **Histórico de Envio** | Acompanha cada e-mail disparado, com status de entrega, falhas e opção de reenvio. |
| **Webhooks** | Endpoints externos que são notificados a cada evento de e-mail (enviado, aberto, rejeitado, falha). |

## O que dá pra fazer aqui

### Abrir uma das seções

1. Clique no cartão desejado (**Templates**, **Histórico de Envio** ou **Webhooks**).
2. Você é levado direto para a seção escolhida.

## Dicas e observações

- Esta tela é apenas um menu de navegação; o trabalho de verdade acontece dentro de cada seção.
- Comece pelos **Templates** se for a primeira vez: é neles que você define o conteúdo dos e-mails que o sistema envia.
